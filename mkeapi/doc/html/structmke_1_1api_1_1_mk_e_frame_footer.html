Detailed Description

MkE 3D data frame footer.

The 3D data from the sensor is provided as a reply data payload, which in this context is called a frame. MkE frame data (or 3D data) consists a variable number of data entries (MkEFrameData1, MkEFrameData2, MkEFrameData3, or MkEFrameData4) of the same type and a frame footer. The data entries are in little endian format and the memory layout of the MkE frame data is as follows:

MkEFrameType1 MkEFrameType2 ... MkEFrameType* MkEFrameFooter The 3D Data frame consists of frame items followed by a frame footer.
